# Security policy

## Supported versions

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Latest release tag | Yes |
| Older tags | Best effort |

## Reporting a vulnerability

**Do not** open a public GitHub issue for security vulnerabilities.

1. Use [GitHub private vulnerability reporting](https://github.com/mmccalla/coding-agent-skill-library/security/advisories/new) when enabled for this repository.
2. If private reporting is unavailable, contact the repository owner via GitHub with minimal reproduction detail and avoid posting secrets or exploit payloads in public channels.

Include:

- Affected component (skills content, MCP server, API, UI, Docker stack, CI)
- Steps to reproduce
- Impact assessment
- Suggested fix or mitigation, if known

## Response expectations

- Acknowledgement within **5 business days** where possible
- Triage and severity assessment before public disclosure
- Coordinated disclosure for issues affecting downstream users of the skills library or KG service
