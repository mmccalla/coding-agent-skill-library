# User Research and Usability Testing

## When to use

Use this skill when validating journeys, forms, dashboards, onboarding flows, approval flows, quarantine triage, data-quality investigations, metadata review, agent recommendations or high-risk workflows.

## Objective

Validate whether target users can understand and complete the intended task safely, accurately and efficiently.

## Procedure

1. Identify target user roles.
2. Define the task to test.
3. Define success criteria.
4. Create realistic test scenarios.
5. Ask users to perform the task with minimal guidance.
6. Observe hesitation, misunderstanding, mis-clicks and questions.
7. Record task completion, errors, time, confidence and comments.
8. Prioritise findings by severity and frequency.
9. Convert findings into design or implementation changes.
10. Re-test the highest-risk changes.

## Methods

Task-based usability test, cognitive walkthrough, first-click test, five-second test, interview, prototype review and accessibility review.

## Rules

- Do not ask users whether they “like” the design as the primary measure.
- Do not test only with builders if the target user is operational or business-facing.
- Do not treat one user's preference as universal truth.
- Do not ignore severe errors because the average score is acceptable.
- Prefer observed behaviour over stated preference.
