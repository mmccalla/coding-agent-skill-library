# KRAG Systems

Use this category when the task is mainly about designing, building, evaluating or governing an end-to-end Knowledge Graph-Augmented Retrieval system.

## Start here when

- the graph must materially improve retrieval, reasoning, provenance or explainability;
- you need evidence-backed graph ingestion, graph-native retrieval or KRAG evaluation gates;
- the work is broader than a single ontology or a single Neo4j query pattern.

## Skills in this category

- `krag-system-design`
- `krag-ingestion-graph-construction`
- `krag-retrieval-answering`
- `krag-evaluation-governance`

## Choose the smallest fit

- Start with `krag-system-design` for architecture, graph role and thin-slice planning.
- Use `krag-ingestion-graph-construction` for ingestion, extraction, evidence anchoring and graph validation.
- Use `krag-retrieval-answering` for retrieval planning, ranking, answering and abstention.
- Use `krag-evaluation-governance` for quality gates, observability, governance and release readiness.
