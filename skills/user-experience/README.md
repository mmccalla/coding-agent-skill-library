# User Experience

Use this category when the task is mainly about user-facing workflows, interfaces, evidence presentation or interaction quality.

## Start here when

- the work changes a screen, flow, form, dashboard or interaction state;
- accessibility, usability or design consistency matters;
- users must supervise, approve or audit agent behaviour.

## Skills in this category

- `ux-design-principles`
- `accessibility-wcag`
- `ui-component-design`
- `frontend-state-and-interaction-design`
- `data-product-dashboard-design`
- `design-system-practice`
- `user-research-and-usability-testing`
- `agentic-ux-patterns`

## Choose the smallest fit

- Start with `ux-design-principles` for journeys and workflow clarity.
- Use `accessibility-wcag` for every user-facing change.
- Use `frontend-state-and-interaction-design` when system states are the hard part.
- Use `agentic-ux-patterns` when people supervise or approve AI actions.
