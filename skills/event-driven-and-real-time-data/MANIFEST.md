# Event-Driven and Real-Time Data Skills Manifest

Use these skills for practical event-driven architecture and real-time data.

| Skill | Use when |
|---|---|
| `event-driven-architecture` | Designing asynchronous event-first systems. |
| `event-modeling` | Discovering business events, commands and state changes. |
| `event-streaming-platform-design` | Designing shared streaming platforms, topics, partitions, retention and replay. |
| `schema-registry-and-contracts` | Defining event schemas, compatibility and producer/consumer contracts. |
| `cdc-and-source-to-stream-ingestion` | Designing CDC and source-to-stream ingestion. |
| `stream-processing-patterns` | Designing enrichment, joins, windows, aggregation and stateful processing. |
| `event-governance-and-lineage` | Governing event ownership, classification, metadata, lineage and lifecycle. |
| `streaming-operations-and-slos` | Designing lag, freshness, replay, back-pressure, SLOs and streaming operations. |
