# Capability Maturity Assessment

## When to use
Use when prioritising capability improvement.

## Objective
Produce a practical, concise, traceable architecture artefact that a coding agent can use to guide implementation or review.

## Procedure
1. Define scale.
2. Gather evidence.
3. Score current maturity.
4. Define target maturity.
5. Identify gaps and risks.
6. Assess importance/urgency.
7. Prioritise actions.
8. Define roadmap increments.

## Required outputs
- Maturity scale
- Current/target scores
- Evidence
- Gaps and risks
- Prioritised roadmap

## Best-practice alignment
Keep capabilities, value streams, processes, organisation, information concepts, initiatives and metrics separate, then link them from strategy to execution.

## Quality checks
- Scores are evidence-based.
- Not every capability targets highest maturity.
- Priorities explain value/risk.

## Avoid
Do not score maturity by opinion alone.

## Verification
State artefacts produced, decisions made, assumptions, risks, validation performed and files changed.
