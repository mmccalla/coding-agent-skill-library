# Enterprise Integration Patterns Skills Pack

A compact, tool-agnostic skills pack for coding agents designing, reviewing and implementing message-based enterprise integrations.

Source basis: Enterprise Integration Patterns by Gregor Hohpe and Bobby Woolf, especially the Messaging Patterns catalogue.

Attribution: Pattern names, problem/solution concepts and pattern taxonomy are derived from Enterprise Integration Patterns, https://www.enterpriseintegrationpatterns.com/patterns/messaging/.

## Skills

| Skill | Purpose |
| --- | --- |
| `message-based-integration-design` | Select integration style and topology. |
| `message-channel-design` | Design channels, delivery guarantees and channel boundaries. |
| `integration-message-construction` | Define message intent, structure, identifiers and lifecycle metadata. |
| `message-routing-design` | Route, split, aggregate, resequence and orchestrate message flows. |
| `message-transformation-design` | Translate, enrich, filter and normalise payloads. |
| `message-endpoint-design` | Implement producers, consumers, gateways and receiver semantics. |
| `messaging-system-management` | Observability, testability, control and operational safety. |
| `eip-integration-validation` | Review designs against EIP, reliability, security and operability criteria. |

## Use

Use one skill at a time when retrieval precision matters. Use `eip-integration-validation` after any design or implementation work.

Category membership is recorded in `skills/PACK_METADATA.json` under `enterprise-integration-patterns` and inventoried in `skills/MANIFEST.md`.
